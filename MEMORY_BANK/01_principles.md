# Законы проекта (Principles)

1) **Контекст важнее кода**: Memory Bank и контракты первичны.
2) **Ноль фантазии**: если данных нет, фиксируем вопрос/риск.
3) **Идемпотентность по умолчанию**: безопасные ретраи.
4) **Детерминизм**: тесты и симуляции должны воспроизводиться.
5) **Наблюдаемость**: логи, коды ошибок, метрики.

## LAW: CONTROL PLAYER FATIGUE (НЕ СПОРИМ ВКУСАМИ — ДАЕМ ВЫБОР)

1) Любая потенциально раздражающая/вкусовая анимация (bounce, squash&stretch, вторичные пост-эффекты) **обязана** иметь переключатель.
2) **Должны существовать 2 режима**:
   - **Reduce Motion**: отключает раздражающие движения и сокращает вторичные эффекты.
   - **Turbo Spin**: минимальный цикл спина без декоративных пост-спин анимаций.
3) Если в артефактах нет явного поведения для Reduce Motion/Turbo Spin — агент **НЕ ИМЕЕТ ПРАВА** импровизировать.
